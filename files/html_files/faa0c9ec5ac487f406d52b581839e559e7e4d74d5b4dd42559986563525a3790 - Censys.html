Censys supports both full-text searches and structured queries. The search Dell will find any hosts where the word Dell appears. However, you can also compose more complex queries. For example, the query ip:192.168.0.0/24 AND metadata.manufacturer:"Dell" will find any Dell devices in the specified network.

Boolean Logic

You can compose multiple statements using AND, OR, NOT, and parentheses. For example, ("Schneider Electric" OR Dell) AND ip:192.168.0.0/24. By default, all terms are optional (e.g., executed as an or statement) unless specified otherwise.

Ranges

You can search for ranges of numbers using [ and ] for inclusive ranges and { and } for exclusive ranges. For example, 80.http.get.status_code:[200 TO 300]. You can search for IP addresses using CIDR notation, e.g., ip:192.168.0.0/16. Timestamps should be queried using the following syntax: [2012-01-01 TO 2012-12-31].

Wildcards

By default, Censys will search for complete words. In other words, the search Del will not return results with the word Dell. If you want to search for words that start with Del, you would search for Del*. You can also search for D?ll.

DNS Queries

You can perform inline DNS A and MX queries using the following syntax: a:facebook.com and mx:gmail.com.

Regular Expressions

You can also search using regular expressions, e.g., metadata.manufacturer:/De[ll]/. Full syntax is available here.

Boosting

The boost operator (^) can be used to make one term more relevant than another. For example, metadata.manufacturer: Dell^2 OR Schneider Electric puts more preference on the Dell keyword.

Reserved Characters

The following characters must be escaped with a backslash: + - = && || > < ! ( ) { } [ ] ^ " ~ * ? : \ /

Censys is a search engine that provides access to three datasets: public IPv4 hosts, websites in the Alexa Top Million Domains, and known X.509 certificates. We provide several ways of accessing this data, including full text search, structured search, a SQL interface, and the ability to download raw data. All interfaces are accessible through both the web interface and a RESTful API.

Full Text Search

Full text search supports simple searches for a word or phrase anywhere in a document. For example, searching Google will return hosts that have the word "Google" in any field. Censys also supports regular expressions. For example, /D.ll/ will match "Dell", "dull", "doll", etc.

Structured Search

Censys supports advanced queries against the structured fields we derive during a scan. For example, you can search for hosts with the HTTP Server Header "Apache" by running the query 80.http.get.headers.server: Apache or for hosts with browser trusted HTTPS certificates with the query 443.https.tls.validation.browser_trusted: true. The data we collect through scans is structured by port, protocol, and then "subprotocol"—the measurement of how the protocol is configured–in this case a GET / request and a TLS handshake, respectively. A list of defined fields are available: IPv4, websites, and certificates.

Where possible, we annotate higher-level information such as server software, geographic location, autonomous system, and device identity with ZTag. The top-level metadata fields describe the device as a whole and each protocol's metadata describes information extracted about that protocol's server-side software. Details on how we tag devices can be found in our research paper.

You can combine multiple searches using AND, OR, and NOT, e.g., 443.https.tls.validation.browser_trusted: true AND 80.http.get.headers.server: Apache. Full details on the search syntax are available here. More advanced examples are available at the bottom of this page.

SQL Query Engine

While the search interface lets you construct many structured queries, not all questions can be formulated using these tools. To facilitate answering more complex questions, it is possible to run SQL queries against current and historical scan data. We are currently restricting SQL access to verified researchers. Please contact us for access.

RESTful API

We provide a RESTful API that facilitates programmatic access to all of the functionality available in the web front end. Please see our API Documentation for details.

Raw Data

We provide access to all our raw data in JSON form, which can be downloaded from Raw Data. These interface also describes the scans we regularly run and the latest results.

Building Reports

Censys also supports generating reports on aggregate statistics. For example, if you wanted to generate a report on cipher suites used by HTTPS sites with browser certificates, you could query 443.https.tls.validation.browser_trusted: true, then generating a report by going to Tools &rarr Build Report and generating a report on the breakdown of the field 443.https.tls.cipher_suite.name.

In many cases, reports should be generated via the .raw version of the appropriate field. This causes the entire field value to be used as a "bin" instead of each individual word. For instance, generating a report on the field location.country results in separate bins being created for "united" and "states" where as the location.country.raw field appropriately creates a single bin "United States".
